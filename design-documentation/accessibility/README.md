# Accessibility

## Our Commitment

Tip Share Pro is committed to providing an accessible experience for all users, regardless of ability. We design and develop with accessibility as a core requirement, not an afterthought.

---

## Target Compliance

**WCAG 2.1 Level AA** is our baseline standard, with Level AAA compliance where practical.

- [Accessibility Guidelines](./guidelines.md) — Complete technical accessibility specifications

---

## Key Principles

### 1. Perceivable

All information and UI components must be presentable in ways users can perceive:

- High contrast color palette (verified ratios)
- Multiple ways to understand information (not just color)
- Text alternatives for non-text content
- Scalable text that works at 200% zoom

### 2. Operable

All functionality must be available to all users:

- Full keyboard navigation
- No keyboard traps
- Skip navigation links
- Adequate time for interactions
- No content that causes seizures

### 3. Understandable

Information and UI operation must be understandable:

- Clear, consistent navigation
- Predictable behavior
- Input assistance and error prevention
- Clear labeling and instructions

### 4. Robust

Content must work across diverse user agents:

- Valid HTML structure
- Proper ARIA usage
- Screen reader compatibility
- Works with assistive technologies

---

## Verification

We verify accessibility through:

1. **Automated testing** with axe-core and Lighthouse
2. **Manual keyboard testing** for all flows
3. **Screen reader testing** with NVDA, VoiceOver, and TalkBack
4. **Color contrast verification** for all color combinations
5. **Reduced motion testing** for animation preferences
